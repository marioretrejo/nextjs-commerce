2. WORKER DE IA (agent/worker.ts) — Enterprise Intelligence Layer
──────────────────────────────────────────────────────────────────────────────
[✅] Pilar 1 — Dynamic Context Injection
       Reemplaza {{contact_name}}, {{company}}, {{phone}} etc. en prompts
       con datos reales del contacto/campaña antes de iniciar la llamada.

[✅] Pilar 2 — HARD CONSTRAINTS Guardrails
       Bloque de sistema prepend: anti-alucinación, resistencia a jailbreak,
       enforced voice format (sin markdown), identidad del agente.

[✅] Pilar 3 — Knowledge Base RAG (Active RAG)
       Tool search_knowledge_base: embed query → pgvector match →
       devuelve los 4 chunks más relevantes. Modelo: text-embedding-3-small.

[✅] Pilar 4 — Tool Calling Guidance
       Sección TOOLS_GUIDANCE dinámica que le explica al LLM cuándo y
       cómo usar cada tool disponible (transfer, RAG, custom tools, end_call).

[✅] Pilar 5 — AI State Machine (flow_config v2) ← NUEVO
       - buildStateMachine(): closure mutable con currentNodeId
       - buildStateInstructions(): concatena base prompt + instrucciones
         del nodo actual + sección "## State Transitions" con intent_ids
       - transition_state tool: valida intent_id, maneja 4 tipos de nodo:
           · ai_state / semantic_router → actualiza instrucciones LLM
           · end_call_node → farewell + deleteRoom
           · transfer_node → SIP REFER passthrough
           · webhook_node → HTTP POST + advance al siguiente nodo
       - agentRef pattern: actualiza agent.instructions mid-call sin reiniciar

[✅] STT: Deepgram nova-3, multilenguaje, redacción de PII (PCI/SSN/numbers)
[✅] TTS: Cartesia sonic-3 con emotion map → OpenAI TTS fallback
[✅] LLM: Groq llama-4-scout → OpenAI gpt-4o-mini FallbackAdapter
[✅] Backchanneling: "Mhm", "I see" durante speech largo del usuario (>3.2s)
[✅] Filler Suppression: "uhm", "er", "yeah" no disparan respuesta LLM
[✅] Human Transfer: SIP REFER vía LiveKit RoomServiceClient
[✅] Dynamic Tools: carga agent_tools desde DB, ejecuta HTTP endpoints custom
[✅] Pronunciation Dicts: keywords Deepgram + phonetic map Cartesia desde DB
[✅] Distributed Tracing: latencia STT/LLM/TTS con threshold alerts
[✅] Mid-call balance check: kill switch cada 60s si se agotan los minutos
[✅] Transcript acumulación: guardado en calls.transcript al cerrar sesión
[✅] Flow v1 coexistencia: buildFlowPrompt() para flow_json IVR clásico

PRIORIDAD MEDIA — Mejoras importantes al producto
──────────────────────────────────────────────────────────────────────────────

[❌] LEAKED PASSWORD PROTECTION en Supabase
       Activar manualmente: Authentication → Password → Enable HaveIBeenPwned.
       No hay API para esto, se hace desde el Dashboard.

[❌] METADATA flow_config en OUTBOUND CALLS
       El worker carga flow_config desde DB como fallback, pero las llamadas
       outbound deberían inyectar flow_config en el room.metadata desde la
       ruta de dispatch para evitar la segunda query a Supabase.

[❌] WEBHOOK NODE — Variables extraídas
       El campo extract_variables del webhook_node está definido pero no
       parseado. El resultado del webhook debería mergear variables en
       el contexto del agente para que el LLM las pueda referenciar.

[❌] SEMANTIC ROUTER — Fallback de intent
       Si el LLM llama transition_state con un intent_id no válido, el error
       actual es informativo pero no hay un nodo de fallback configurable.
       Añadir soporte para un edge con id "fallback" o "default".

[❌] FLOW BUILDER — Validación antes de guardar
       No valida que el flujo tenga al menos start_node → ai_state → end_call_node
       antes de guardar. Un flujo malformado puede romper el worker silenciosamente.

[❌] FLOW BUILDER — Preview / simulación
       No hay forma de simular una conversación en el flow builder antes de
       publicarlo. Sería valioso para testing sin hacer llamadas reales.

[❌] CAMPAÑA → flow_config
       Las campañas outbound usan el prompt del agente directamente.
       Permitir seleccionar un flujo específico por campaña (override del
       flow_config del agente para esa campaña).

[❌] INTEGRACIÓN HUBSPOT REAL
       La carpeta lib/hubspot existe pero la sincronización post-call
       (crear contacto + activity en HubSpot tras analyze-call) no está
       conectada al dispatcher. Solo está el skeleton.

[❌] INTEGRACIÓN SALESFORCE / GOHIGHLEVEL
       Similar a HubSpot — estructura existe pero dispatch no implementado.

[❌] VOICE CLONING

[❌] REAL-TIME DASHBOARD (WebSocket)
       El dashboard actual hace polling. Con LiveKit Data Channels o Supabase
       Realtime se podrían mostrar llamadas activas en vivo.

[❌] MOBILE APP
       Todo el producto es web. Una app móvil (React Native) para supervisores
       que quieran monitorear campañas desde el teléfono.

[❌] EXPORTACIÓN DE REPORTES
       No hay export a CSV/PDF de analytics, transcripts o llamadas.
       Útil para compliance y reporting comercial.

[❌] MULTI-IDIOMA EN UI
